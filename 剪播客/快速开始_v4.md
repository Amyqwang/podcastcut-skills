## ⚡ 3步快速使用

### 1️⃣ 准备信息

**必需输入**：
- 音频文件路径：`/path/to/your/podcast.mp3`
- **说话人数量**：`2`（或3、4等）— **必须明确提供**
- 说话人姓名：麦雅、响歌歌、安安（可选）

**⚠️ 重要提醒**：
- Claude **不会自行判断**说话人数量
- 如果你没有提供，Claude会先询问你
- 说话人数量错误会严重影响准确度（98.8% → 低准确度）

**示例**：
```
音频：/path/to/your/原始音频.mp3
说话人：3人（麦雅、响歌歌、安安）
```

---

### 2️⃣ 配置API Key

```bash
cd $PODCASTCUT_DIR

# 如果没有.env文件，创建它
cat >> .env << 'EOF'
DASHSCOPE_API_KEY=sk-your-api-key-here
EOF

# 获取API Key: https://dashscope.console.aliyun.com/
```

---

### 3️⃣ 告诉Claude

```
帮我剪这个播客：
- 音频：/path/to/your/原始音频.mp3
- 说话人：3人（麦雅、响歌歌、安安）
```

或简单地说：
```
剪播客 /path/to/audio.mp3，3个说话人
```

---

### 4️⃣ Claude自动执行

Claude会自动：
1. 创建输出目录
2. 准备音频文件
3. 上传到公网（uguu.se）
4. **转录+说话人映射**（3分钟）
   - 调用阿里云API转录
   - 识别说话人身份
   - 生成字级别转录 (subtitles_words.json)
5. Claude深度语义分析
6. 生成增强审查界面
7. 启动本地服务器

---

### 5️⃣ 审核和剪辑

1. 打开审查界面：http://localhost:8849
2. 查看AI建议的删除标记
3. 调整确认
4. 点击"执行剪辑"
5. 获得精剪版：`3_成品/播客名_精剪版_v1.mp3`

## 🎯 关键参数

### 说话人数量（必需！）

**如何确定**（由用户判断）：
- 听前2-3分钟音频，数一数有几个不同声音
- 或查看节目大纲/描述
- 计算公式：`主播数 + 嘉宾数 = 总人数`

**示例**：
- 单人播客（独白）→ `1`
- 双人播客（对话）→ `2`
- 2主播+1嘉宾 → `3`
- 圆桌讨论4人 → `4`

**重要**：
- 设置错误会导致说话人识别不准（98.8% → 低准确度）
- Claude不会自行判断，必须由你提供
- 如果不确定，听一下音频开头1-2分钟再决定

## ❓ 常见问题

### Q1: 说话人识别不准确？

**检查**：
1. `speaker_count`参数是否正确？
2. 音频质量是否清晰？
3. 说话人声音是否有明显差异？

**解决**：
- 重新转录，设置正确的说话人数量
- 或使用本地FunASR（准确度更高）
- 差异通常<2%，人工校对工作量很小

### Q2: uguu.se链接过期？

uguu.se文件24小时后自动删除。

**解决方案**：
- 使用阿里云OSS（长期有效）
- 使用七牛云、腾讯云COS
- 使用自己的服务器

### Q3: API成本多少？

阿里云FunASR按音频时长计费：
- 查看最新价格：https://help.aliyun.com/zh/model-studio/
- 2小时播客约¥X（查询官网）

### Q4: 可以用本地FunASR吗？

可以！但需要：
1. 修改skill使用本地脚本
2. 安装FunASR依赖
3. 首次下载模型（944MB）

**trade-off**：
- 速度慢7倍（20分钟 vs 3分钟）
- 但完全免费
- 准确度稍高（99%+ vs 98.8%）

## 🎬 完整命令参考

### 手动执行（高级用户）

```bash
# 0. 变量设置
AUDIO_PATH="/path/to/podcast.mp3"
SPEAKER_COUNT=3
SKILL_DIR="$PODCASTCUT_DIR"

# 1. 创建目录
AUDIO_NAME=$(basename "$AUDIO_PATH" .mp3)
BASE_DIR="output/$(date +%Y-%m-%d)_${AUDIO_NAME}/剪播客"
mkdir -p "$BASE_DIR/1_转录" "$BASE_DIR/2_分析" "$BASE_DIR/3_成品"

# 2. 准备音频
cd "$BASE_DIR/1_转录"
cp "$AUDIO_PATH" audio.mp3

# 3. 上传获取URL
AUDIO_URL=$(curl -s -F "files[]=@audio.mp3" "https://uguu.se/upload?output=text")
echo "$AUDIO_URL" > audio_url.txt

# 4. 转录 + 说话人映射 → subtitles_words.json（~3分钟）
export DASHSCOPE_API_KEY="sk-your-key"

# 4a. API转录
bash "$SKILL_DIR/剪播客/scripts/aliyun_funasr_transcribe.sh" "$AUDIO_URL" "$SPEAKER_COUNT"

# 4b. 识别说话人
node "$SKILL_DIR/剪播客/scripts/identify_speakers.js" aliyun_funasr_transcription.json

# 4c. 创建映射（根据4b的输出）
cat > speaker_mapping.json << 'EOF'
{"0":"麦雅","1":"响歌歌","2":"安安"}
EOF

# 4d. 生成字级别转录
node "$SKILL_DIR/剪播客/scripts/generate_subtitles_from_aliyun.js" \
  aliyun_funasr_transcription.json \
  speaker_mapping.json

# 5. 后续交给Claude分析和剪辑
```
